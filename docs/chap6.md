# ドメインの完全性と整合性

- 完全性
    - ドメイン内の個々の情報がそのドメイン内でのルールを満たしていること
    - 例:ユーザーIDは英数字7桁、重量は0 ~ 1000の整数 など
- 整合性
    - ドメイン内の情報間でそのドメイン内でのルールを満たしていること
    - 例:ある口座から別の口座に送金 -> 送金元の口座が同額減り送金先は同額増えている
- 保つ方法
    - 完全性
        - コンストラクタをprivateに + 検証を行うメソッドを定義
    - 整合性
        - 集約の頂点となるオブジェクトで管理する
        - 集約は原子性の単位でもある
            - 1トランザクションで更新するのは1集約
            - ワークフローを一つのトランザクションとして見る場合、集約を束ねた集約を作成しても良い（その場合責任はその集約に改めて移る）
        - 例:注文の合計金額は個別の商品の額の合計に等しい という整合性
            - 注文に個別の商品の注文個数が集約されている
            - なので集約の頂点である注文が整合性を維持する
- ビジネスルールを型で表現する
    - ビジネスルール上存在すべきではない状態を型システムで防止する
        - 例:ユーザーのメールアドレスは検証済みのみ
            - 未検証と検証済みで別の型として定義することで不正な状態を作らせない
        - 例:二つの情報のうちどちらかが必要
            - それぞれの情報を片方または両方持つ状態を表す型を作成する
- 結果整合性は全てのコンテキスト間で整合性をとるより実装コストが低い
