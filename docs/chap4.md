# 型の理解
ドメインのモデリングを関数型の手法で実装

- 用語の定義
    - 関数
        - 入力と出力を持つブラックボックス
    - 型
        - 関数の入力または出力として使用可能な値の集合に与えられた名称
        - 関数も値として利用できる
            - int -> (int -> int)
                - int型の値を受け取り、(int -> int)という型シグネチャの関数を返す という関数の型シグネチャ
- F#での関数
    - 型シグネチャという記法で表される
        - 例:
            - let add1 x = x + 1
                - int -> int
            - let add2 x y = x + y
                - int -> int -> int
        - 各パラメータは矢印で区切られる
    - 定義
        - letキーワードを使って定義
        - 返り値は明示的に書かなくていい
        - 関数の中にサブ関数を定義できる
        - ジェネリック型を利用できる
            - F#では型の名前に'がつく
    - 値の定義と型の定義が似ている
    - ケース名
        - 型そのものではなく、その型に属する別のケース名
        - 選択型の場合どちらもその型
- 型の合成
    - 複数の型を合成して異なる型を作成する
    - 合成の種類
        - AND型（直積型）
            - 内部に複数の型を持つオブジェクト
        - OR型（直和型）
            - 複数の型のうちどれか一つを用いる
            - F#では判別共用体
        - 単純型
            - 既存の型を別の型として定義する
        - 代数的な型システム
            - 全ての複合型がより小さな型をANDまたはORで合成してできているもの
- 型の合成によるモデリング
    - 型を使ったモデリング
        - 上の型の合成などのと技術を使いモデリングを行う
        - 低レベルな型の定義    
            - プリミティブな型のラッパーとして定義
            - 各ワークフローの入力をわかりやすくする
        - 合成の型
            - レコード型で複数の型の値を集めたものを表現
            - OR型でいずれの型かを選択
        - 関数の型
            - ワークフローの入出力を型で表す
    - 省略可能な値、エラー、コレクションのモデリング
        - 省略可能な型
            - option型を利用
        - エラー
            - Result型を利用
            - 成功とエラーを型で明示的に表したい時、Result型で成功時失敗時の型を指定する
        - 値を返さない関数
            - unit型を利用
                - 他言語でいうvoid型相当
            - 関数内で副作用があることを強く示唆する
                - 基本的に値の型間での変換を記述していくのが関数型言語での実装である -> 型の変換を行わない（値を返さない）のであれば書く必要はないはず -> あえて書いているのは型の変換以外のことを記述する必然性がある -> 副作用を記述していると考えられる（ワークフローの出力は値と副作用のため）
