# 関数型アーキテクチャ

- ソフトウェアアーキテクチャにおけるユビキタス言語としてC4アプローチの語彙を利用する
    - システムコンテキスト
        - システム全体を表す最上位の概念
        - 複数のコンテナから構成される
    - コンテナ
        - デプロイ可能な単位
        - 複数のコンポーネントから構成される
    - コンポーネント
        - コードの構造において主要な概念
        - 複数のクラスから構成される
    - クラス（モジュール）
        - 低レベルのモジュールや関数の集合が含まれる
- 優れたアーキテクチャの設計 -> 上記概念間の境界を変更コストを小さくするように定義すること
- 境界づけられたコンテキスト間のコミュニケーション
    - イベントを使って行う
    - キューでの通信を用いると完全に疎結合になる
    - Data Transfer Objectを用いて通信する
        - ドメイン型 -> DTO -> ドメイン型
    - 各ワークフローの先頭+末端でゲートと呼ばれる検証プロセスを設ける
        - 先頭
            - DTO -> ドメイン型の変換後に制約を満たしているかを検証する
            - 満たしていない場合はエラー生成しワークフローの実行をスキップ+エラーに対応したイベントを生成
        - 末端
            - 境界づけられたコンテキスト内のプライベートな内容を他コンテキストに開示しないようにDTOに変換する
            - セキュリティの観点＋コンテキスト間の意図しない依存関係の発生を避けるため
- 境界づけられたコンテキスト間の契約
    - 二つのコンテキスト間のコミュニケーションが発生する->コンテキスト間の契約が発生する
        - 二つのコンテキスト間でコミュニケーションのための共通フォーマットに合意する
    - コンテキスト間の関係を契約の詳細を決定する主体に基づいて分類
        - 共有カーネル
            - 二つのコンテキストが共通のドメイン設計を共有している関係
            - 両コンテキスト間で相談してから変更する必要がある
        - 顧客+供給者
            - 下流である顧客側が定義した義務に上流である供給者が従う限り独立して発展できる関係
            - 下流コンテキストが決定し上流がそれに従う
        - 順応者
            - 供給者+顧客の逆
            - 上流のコンテキストが決定し下流がそれに従う
    - ゲート内での変換＋検証によりコンテキスト内のモデルが他コンテキストに影響を受ける（腐敗）ことを防ぐ
        - 腐敗防止層という役割
- 境界づけられたコンテキストでのワークフロー
    - ビジネスワークフローの対応
        - ドメインの探索プロセス（イベントストーミングなど）
            - 1つか複数個のドメインイベントを生成するコマンドによって開始される一連の流れ
        - 関数型アーキテクチャ
            - 入力がコマンドオブジェクト＋出力がイベントオブジェクトのリストである単一の関数

# 単語の定義
- ドメインオブジェクト
    - ある境界づけられたコンテキスト内でのみ使用するように設計されたオブジェクト
    - DTOとは対照的
- データ転送オブジェクト（DTO）
    - 境界づけられたコンテキスト間で共有されるように設計されたオブジェクト
