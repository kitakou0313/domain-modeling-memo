# 関数型アーキテクチャ

- ソフトウェアアーキテクチャにおけるユビキタス言語としてC4アプローチの語彙を利用する
    - システムコンテキスト
        - システム全体を表す最上位の概念
        - 複数のコンテナから構成される
    - コンテナ
        - デプロイ可能な単位
        - 複数のコンポーネントから構成される
    - コンポーネント
        - コードの構造において主要な概念
        - 複数のクラスから構成される
    - クラス（モジュール）
        - 低レベルのモジュールや関数の集合が含まれる
- 優れたアーキテクチャの設計 -> 上記概念間の境界を変更コストを小さくするように定義すること
- 境界づけられたコンテキスト間のコミュニケーション
    - イベントを使って行う
    - キューでの通信を用いると完全に疎結合になる
    - Data Transfer Objectを用いて通信する
        - ドメイン型 -> DTO -> ドメイン型
    - 各ワークフローの先頭でゲートと呼ばれる検証プロセスを設ける
        - DTO -> ドメイン型の変換後に制約を満たしているかを検証する
        - 満たしていない場合はエラー生成しワークフローの実行をスキップ+エラーに対応したイベントを生成
